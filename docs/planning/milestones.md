# Milestones

This document maps the planned milestones to intended outcomes and related issue groupings. Only Issue #1 currently exists in the repository, so later issue references are intentionally left broad rather than invented.

| Milestone | Intended Outcome | Related Issues |
| --- | --- | --- |
| Milestone 1 — Feasibility Prototypes | Demonstrate whether device-specific input detection and programmatic display switching are viable on the target Windows setup. | `#1` repository foundation/setup; future Raw Input and display-switch prototype issues |
| Milestone 2 — MVP Automatic Switching | Deliver a minimal end-to-end workflow from detected device activity to automatic display profile switching. | Future MVP implementation issues |
| Milestone 3 — GUI and Persistence | Add a usable desktop configuration experience and persisted settings for devices, zones, and profiles. | Future GUI, configuration, and persistence issues |
| Milestone 4 — Robustness and Smart Logic | Reduce false switching, improve recovery behaviour, and introduce smarter switching rules. | Future robustness, cooldown, recovery, and policy issues |
| Milestone 5 — Product Polish | Improve packaging, diagnostics, usability, and day-to-day reliability for regular use. | Future polish, packaging, and operational quality issues |
| Milestone 6 — Stretch / Research | Explore advanced or uncertain capabilities beyond the core product scope. | Future research issues such as lock screen support, service companion work, or more advanced decision logic |

## Notes

- Milestones are outcome-oriented rather than tied to a fixed calendar.
- New issues should be linked to the milestone they most directly advance.
- If feasibility results materially change scope, milestone definitions should be revisited rather than forcing the roadmap to remain unchanged.
